Question: Outline the steps you took to get to your final CNN architecture and your reasoning at each step. Describe why you think the architecture is suitable for the current problem.

Selecting an Already Trained Model:

Step: Decided to use ResNet18 as the transfer learning model that was already trained. Reasoning: In image classification applications, ResNet18 is renowned for its efficiency and robustness. Because of its depth and residual connections, which aid in handling vanishing gradients and learning complicated properties, its architecture is appropriate for transfer learning.

The Pre-trained Model is loaded.

Step: Pre-trained on ImageNet, loaded the ResNet18 model. Reasoning: A good place to start is using a model that has already been pre-trained on a sizable dataset, such as ImageNet. The model can already be adjusted for the particular landmark categorization task because it has already acquired useful features.

Freezing Layers to Extract Features:

Step: To freeze all layers but the last fully connected layer, set feature_extract=True. Reasoning: By only updating the last layer during training, freezing the preceding layers enables the model to take advantage of the generic features that it has learnt from ImageNet, lowering the risk of overfitting. Changing the Last Layer:

Step: A new layer with the right amount of output classes (50 classes) was added to replace the last fully connected layer. Reasoning: ResNet18's initial final layer was created with 1000 classes for ImageNet. The output layer must match the 50 landmark classes in our dataset in order to pass the landmark classification task.

Organizing the Process of Training:

Step: Specified hyperparameters, including learning rate, epoch count, optimizer (Adam), batch size, validation size, and weight decay. Reasoning: Properly selected hyperparameters aid in striking a balance between model performance and training efficiency. A moderate learning rate combined with the Adam optimizer guarantees effective and steady convergence. Putting Data Augmentation in Practice:

Step: Used data augmentation methods like flipping, normalization, and random cropping. Reasoning: By adding additional diversity to the training set, data augmentation helps reduce overfitting and strengthen the model's resistance to changes in input image quality.

Loop for Training and Validation:

Step: Put in place a training loop that runs through each epoch, training and assessing the model on the validation set. Rationale: Frequent assessment of the validation set enables tracking of the model's performance and early termination in the event that the validation accuracy plateaus or the model becomes overfit.

Checkpointing Models:

Step: During training, the model with the best validation accuracy was saved. Rationale: Checkpointing guarantees that the optimal model is preserved, averting the loss of progress owing to any disruptions and permitting additional adjustments or implementation utilizing the top-performing model.

Suitability of the Architecture for the Current Problem

Ready-made ResNet18: The residual connections in ResNet18 facilitate the learning of deep representations without vanishing gradient problems. This enables it to record intricate details that are essential for differentiating between various sites.

Transfer of Learning: We begin with a model that already has a strong capacity to extract generic visual features by utilizing a model that has previously been pre-trained on ImageNet. Its landmark categorization fine-tuning adapts these qualities to our particular goal.

Layers of Freezing: By preserving the generic features acquired from a sizable dataset like as ImageNet, freezing the previous layers contributes to better generalization and lower overfitting risk.

Personalized Last Layer: The output of the model is guaranteed to be immediately applicable to the landmark classification job by modifying the final layer to correspond with the number of classes in our dataset.

Effective Instruction: In contrast to deeper variations like ResNet50, ResNet18's smaller, more efficient architecture strikes a compromise between computational efficiency and performance, making it appropriate for real-world use cases with constrained resources.

Regularization by means of Enhanced Data: By using data augmentation approaches, a model's generalization abilities are enhanced and made more resilient to variances in real-world images.

Selecting Hyperparameters: The model's performance is further improved by setting the selected hyperparameters (learning rate, batch size, optimizer, etc.) to values that encourage steady and effective training.

import torch
from src.train import one_epoch_test

# Define the number of classes
num_classes = 50

model_transfer = get_model_transfer_learning("resnet18", num_classes=num_classes)

# Load saved weights
model_transfer.load_state_dict(torch.load('checkpoints/model_transfer.pt'))

# Ensure the model is in evaluation mode
model_transfer.eval()

# Load data loaders
data_loaders = get_data_loaders(batch_size=1)

# Define the loss function
loss = get_loss()

assert test_accuracy >= 60, "Test accuracy is below 60%. Please tune your hyperparameters and retrain the model."

from src.predictor import Predictor
from src.helpers import compute_mean_and_std

# First let's get the class names from our data loaders
class_names = data_loaders["train"].dataset.classes

# Then let's move the model_transfer to the CPU
# (we don't need GPU for inference)
model_transfer = model_transfer.cpu()
# Let's make sure we use the right weights by loading the
# best weights we have found during training
# NOTE: remember to use map_location='cpu' so the weights
# are loaded on the CPU (and not the GPU)
model_transfer.load_state_dict(
    torch.load("checkpoints/model_transfer.pt", map_location="cpu")
)

# Let's wrap our model using the predictor class
mean, std = compute_mean_and_std()
predictor = Predictor(model_transfer, class_names, mean, std).cpu()

# Export using torch.jit.script
scripted_predictor = torch.jit.script(predictor)
scripted_predictor.save("checkpoints/transfer_exported.pt")
